# 기획안
## 프로젝트 개요
1. 프로젝트명 : Taste Atlas: Personal Taste Knowledge Map
2. 문제 : 개인의 취향에 대한 정보가 플랫폼별로 흩어져 있어 "취향의 맥락"을 학습하기 어려움
3. 목표: 검색 기반으로 취향 정보를 지도처럼 구조화하여, 아티스트/작품/장르/시대를 연결한 학습형 아카이브를 통해 취향의 맥락을 파악해 자기탐구의 영역을 넓힌다.
4. 형태 : 웹 mvp (현재) -> 추후 앱/3d노드맵 확장


## 목표/범위
1. In Scope
   1) 아티스트/작품 키워드 검색
   2) 정보 패널에서 관련 내용 학습
   3) 노드 저장 및 아카이빙
   4) 연결된 노드를 통해 깊이있는 취향 탐색
2. Out of Scope
   1) 완전 자동 추천 엔진 (?)
   2) 저작권 이슈가 있는 원문 대량 수집
   3) Spotify OAuth 기반 개인화 데이터 연동
3. 핵심 사용자
   - 취향을 깊이있게 탐색하고자 하는 니즈를 소유한 사용자 (큐레이션 소비자)
   - 콘텐츠 리서치를 위한 실무자 (생산자)
4. 사용자 과업
   -
5. 정보구조
6. 데이터 전략
7. UX 원칙
8. 성공지표 (사용성 테스트를 통한 확인절차)
9. 일정
   - Task Success: 검색→선택→정보 확인 완료율
   - Efficiency: 첫 결과 선택까지 시간, 원하는 정보 도달 시간
   - Satisfaction: 정보 신뢰도/가독성 주관 평가
   - Retention: 재방문 시 노드/아카이브 활용률
10. 리스크/대응
